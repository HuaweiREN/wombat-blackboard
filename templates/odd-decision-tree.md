# ODD 决策树

## 场景描述

本模板用于描述 ADAS/AD 功能的运行设计域（ODD）决策逻辑，以分层决策树的方式判断功能在当前行车条件下是否可用。决策树的根节点为 ODD 总入口，判断流程按优先级依次检查天气条件、道路类型、速度范围、车道线质量、光照条件和地理围栏，每层均为二元或多元分支。只有当所有条件均满足时，功能标记为 Available（可用）；任意条件不满足则标记为 Unavailable（不可用）或 Degraded（降级可用）。

天气条件子决策树是第一道关卡：晴朗天气直接通过；降雨天气进入强度分支，小雨（雨量 < 2.5 mm/h）限制最高速度但功能可用，中雨（2.5-7.5 mm/h）仅允许基础 ACC 功能，大雨及以上禁止所有高级功能；降雪/冰雹天气直接返回不可用；雾天进入能见度分支，能见度 > 200m 功能正常，100-200m 降级运行，< 100m 禁止激活。此判断依赖雨量传感器、光雨量传感器和气象预报数据的融合结果。

道路类型决策分支依据地图数据和道路感知结果综合判断：高速公路（结构化道路、封闭出入口）允许 HWP/NOD 功能；城市主干道（有物理隔离、信号灯控制）允许 NOD 城市版（限城市领航功能），需满足车道线清晰且无行人密集区域；匝道/立交桥区域对 HWP 功能限制速度（< 80 km/h）但允许；乡村道路、居民区道路返回功能不可用。速度范围决策根据当前自车速度和环境限速综合衡量：0-60 km/h 区间允许 TJP 功能；60-130 km/h 区间允许 HWP 功能；130 km/h 以上返回不可用。如有路段临时限速，功能需在进入限速区域前完成降级或退出。

附加判断层包括车道线质量（清晰、模糊、缺失三级判定，影响横控功能可用性）、光照条件（白天、夜晚、黄昏、隧道入口/出口，影响视觉感知置信度）和高精地图覆盖区域（已测绘区域可用，未覆盖区域降级或退出）。当所有决策层级均通过时，系统输出 Available 信号并允许功能激活。当决策树返回 Degraded 时，系统仅启用部分功能或限速运行，同时向 HMI 发送降级提示和接管建议。当返回 Unavailable 时，系统阻止激活或在已激活状态下规划安全退出的过渡策略。

在生成图表时，请使用 Mermaid flowchart 自顶向下展示完整的决策树结构。顶层是总入口节点，逐级向下分支。每个决策节点使用菱形判断框，结果节点使用圆角矩形（Available / Degraded / Unavailable）。建议使用绿色/黄色/红色分别标记可用/降级/不可用结果路径以增强可读性。每条分支线标注判断条件和阈值。

## 关键要素

- 一级判断层: 天气条件、道路类型、速度范围、车道线质量、光照条件、地理围栏的决策优先级
- 天气子树: 晴/雨/雪/雾四种气象条件的分支判断逻辑和量化阈值
- 道路类型子树: 高速公路/城市主干道/匝道/乡村道路的可用性判定，结合地图数据
- 速度子树: TJP / HWP / 不可用三段速度区间划分及与功能的映射关系
- 降级策略: Degraded 模式下的功能裁剪方案（仅纵向/限速/限制变道）
- 多源输入: 雨量传感器、光雨量传感器、地图定位、视觉车道线检测、气象预报

## 参数占位

- {function_name}: TJP / HWP / NOD / ACC，受影响的功能名称
- {max_speed}: 60~130，功能允许的最高速度（km/h）
- {min_speed}: 0~30，功能启用的最低速度（km/h）
- {road_types}: 高速公路 / 城市主干道 / 所有道路，适用的道路类型列表
- {require_lane_marking}: true/false，是否需要车道线清晰可见
- {allow_night}: true/false，是否允许夜间运行
- {weather_limits}: 晴天/小雨/中雨/无雪禁雾，允许的天气条件列表
