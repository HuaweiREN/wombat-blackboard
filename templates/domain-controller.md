# 域控架构

## 场景描述

本模板用于描述智能驾驶域控制器的内部硬件和软件分层架构。典型的域控制器采用异构多芯架构，包含感知计算平面、规划决策平面和安全控制平面三层。感知计算平面由 1-2 颗高性能 SoC 构成（如 Orin-X / J6 / TDA4 VH），负责传感器数据处理、目标识别、多传感器融合和语义地图构建。规划决策平面运行在 SoC 内部或独立的高性能处理器上，负责行为预测、路径规划和决策输出。安全控制平面由 ASIL-D 级功能安全 MCU（如 TC3xx / S32K3 / R-Car S4）构成，负责车辆纵向和横向的运动控制执行以及功能安全监控。

芯片间互联方面，感知 SoC 和安全 MCU 之间通过 PCIe 高速总线进行大容量数据交换（典型 4-8 GT/s），SPI 用于控制命令和状态上报通道。SoC 之间的核间通信通过共享内存和 RPMsg 协议实现。MCU 内部的多核之间通过锁步模式实现故障检测，满足 ASIL-D 的功能安全要求。外部存储配置包括 LPDDR5（SoC 域，带宽 200+ GB/s）、DDR4（MCU 域）以及 QSPI NOR Flash 用于启动代码和固件存储。

外部接口方面，域控提供 8-16 路 GMSL2/3 FPD-Link III SerDes 接口连接摄像头模组，4-6 路 CAN-FD 接口连接车身动力底盘总线，2-4 路 100BASE-T1 / 1000BASE-T1 车载以太网接口连接域间骨干网络。此外提供 1-2 路 PCIe 扩展接口用于未来传感器扩展（如激光雷达直连），GPIO/AIN 接口用于硬线信号采集（刹车踏板、方向盘转角、轮速脉冲等）。电源管理部分采用多路 PMIC + LDO 方案，支持 ASIL-B 等级的供电安全，包含多个独立电压域以实现低功耗待机和快速唤醒。

在生成图表时，请以分层方框图展示域控制器的内部模块划分和芯片间互联关系。上方展示感知计算域的 SoC 集群及其外设连接（摄像头、雷达、激光雷达），中间展示软件功能分配（感知、融合、预测、规划、控制），下方展示安全 MCU 及其执行器连接（转向、制动、动力）。接口方向用箭头标注，数据流和控制流用不同线型区分。

## 关键要素

- 异构多芯架构: 感知 SoC + 安全 MCU 的分工与协作关系
- 芯片间互联: PCIe、SPI、RPMsg、共享内存等芯片间通信协议和带宽
- 外部接口: GMSL/FPD-Link 摄像头接口、CAN-FD 整车总线、车载以太网骨干网
- 存储层次: LPDDR5/DDR4 内存、eMMC/UFS 大容量存储、QSPI NOR Flash 固件存储
- 电源管理: 多路 PMIC 供电域划分、ASIL-B 供电安全、低功耗待机模式
- 功能安全: ASIL-D MCU 锁步、MPU 内存保护、EWM 窗口看门狗、ECC

## 参数占位

- {soc_count}: 1~2，感知 SoC 数量
- {mcu_model}: TC3xx / S32K3 / R-Car S4，安全 MCU 型号
- {has_lidar}: true/false，是否需预留激光雷达数据处理能力
- {adas_level}: L2+ / L3 / L4，所支持的自动驾驶级别
- {camera_ch}: 8~16，摄像头通道数
- {can_fd_ch}: 4~6，CAN-FD 接口数量
- {eth_port}: 2~4，车载以太网端口数
