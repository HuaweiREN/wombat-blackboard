# 功能状态机

## 场景描述

本模板描述 ADAS/AD 功能的状态机设计，适用于 ACC、LKA、TJP、HWP、NOD 等典型智驾功能的状态管理。系统包含六个主要状态：OFF（下电状态）、Standby（待机状态）、Active（激活状态）、Override（超控状态）、Failure（故障状态）和 Limp Home（跛行回家状态）。其中 OFF 状态是系统的初始状态，SOC 上电后完成硬件初始化和自检，满足条件后进入 Standby。Standby 状态下系统处于就绪等待模式，所有传感器和控制器正常供电运行但不介入车辆控制，等待驾驶员发起激活指令或满足自动激活条件后转入 Active。

Active 状态内部包含多级子状态对应不同的功能模式。对于 L3+ 系统，在 Active 状态下进一步划分为 ACC（自适应巡航，不参与横向控制）、LKA（车道保持辅助，纵向横向双控）、TJP（交通拥堵领航，0-60 km/h 全速域）、HWP（高速公路领航，60-130 km/h 高速场景）和 NOD（领航辅助驾驶，含自动超车和匝道汇入）。各子状态之间根据 ODD 变化和驾驶员意图可动态切换，例如车辆驶出高速路段时从 HWP 降级到 TJP 或 ACC。

状态转换的条件和触发机制如下：Standby → Active 需要满足所有 ODD 条件且驾驶员确认（按钮或拨杆）；Active → Override 在检测到驾驶员主动干预（大扭矩转向、刹车或加速踏板超控）时触发；Active → Failure 在系统诊断出不可恢复的硬件或软件故障时触发（如 SoC 过热、摄像头遮挡、IMU 失效）；Override → Active 在驾驶员退出干预后，且 ODD 仍然满足时自动恢复；Override → Standby 在驾驶员持续干预或 ODD 退出后保持超控超过超时阈值（通常 3-5 秒）后进入。Failure 状态分为可恢复和不可恢复两类：可恢复故障（如通信瞬断）在错误恢复后自动回到 Standby；不可恢复故障（如 MCU ECC 错误、PMIC 故障）需要下电重启或到店维修才能清除。Limp Home 作为部分功能的降级模式，在检测到非关键故障时启用，仅提供有限的纵向控制功能，提示驾驶员接管。

在生成图表时，请使用 Mermaid state 图展示完整的状态转换。主状态用方框表示，子状态用复合状态展开。每条转换需标注触发条件（简洁的中文描述）。可使用颜色区分正常流程（绿色）和异常流程（红色）。故障状态的恢复路径需要做特殊标记以体现安全设计的完整性。

## 关键要素

- 六大主状态: OFF、Standby、Active、Override、Failure、Limp Home 的定义和职责
- Active 子状态: ACC、LKA、TJP、HWP、NOD 的功能范围和适用场景
- 转换条件: 驾驶员激活操作、ODD 判断、故障诊断、超控检测、退出超时
- 故障分级: 可恢复故障（瞬态）与不可恢复故障（永久）的差异化处理
- 安全策略: ASIL 分解下的状态管理、冗余监控模块对状态机的交叉校验
- 降级策略: Limp Home 模式下可维持的功能边界（仅纵向、限速 30 km/h）

## 参数占位

- {function_name}: ACC / LKA / TJP / HWP / NOD，目标功能名称
- {has_limp_home}: true/false，是否启用跛行回家降级模式
- {adas_level}: L2 / L2+ / L3 / L4，功能安全等级影响状态转换策略
- {override_timeout}: 3~5，超控超时时长（秒）
- {has_sub_states}: true/false，是否需要在 Active 下展开子状态
